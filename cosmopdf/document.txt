2. ARCHITECTURE & PHILOSOPHY

2.1 STATELESS ARCHITECTURE
--------------------------------------------------------------------------------
CosmoPDF follows a strict stateless design pattern:

- No user accounts required
- No session management
- No database storage
- No server-side processing
- All operations ephemeral (memory-only)

Data Flow:
[User File] → [FileReader API] → [Browser Memory] → [Processing Engine] 
→ [Download] → [Memory Cleared]

2.2 PRIVACY MODEL
--------------------------------------------------------------------------------
The privacy model is built on cryptographic certainty rather than trust:

TRUST-BASED MODEL (Traditional):
  "We promise not to look at your files"

  - Subject to data breaches

CRYPTOGRAPHIC MODEL (CosmoPDF):
  "Files never leave your device"
  - No trust required
  - No remote storage to breach
  - No compliance needed (no data collected)
  - Verifiable via source code inspection

4. TECHNICAL IMPLEMENTATION

4.1 FILE HANDLING
--------------------------------------------------------------------------------
All file operations use the FileReader API:

const reader = new FileReader();
reader.onload = (e) => {
    const arrayBuffer = e.target.result;
    // Process with PDF-LIB
};
reader.readAsArrayBuffer(file);

Memory Management:
- Files stored as ArrayBuffer in RAM
- Automatically garbage collected when:
  * Tab is closed
  * New file uploaded (replaces old)
  * Explicit cleanup called

4.2 PDF PROCESSING
--------------------------------------------------------------------------------
PDF-LIB Operations:

Loading:
  const pdfDoc = await PDFDocument.load(arrayBuffer);

Merging:
  const copiedPages = await mergedPdf.copyPages(sourcePdf, pageIndices);
  copiedPages.forEach((page) => mergedPdf.addPage(page));

Splitting:
  pdfDoc.removePage(index);
  
Rotating:
  page.setRotation(degrees);

Saving:
  const pdfBytes = await pdfDoc.save();
  download(pdfBytes, "output.pdf", "application/pdf");

4.3 RENDERING THUMBNAILS
--------------------------------------------------------------------------------
PDF-JS used for page previews:

const loadingTask = pdfjsLib.getDocument({data: arrayBuffer});
const pdf = await loadingTask.promise;
const page = await pdf.getPage(pageNum);

const viewport = page.getViewport({scale: 0.5});
const canvas = document.createElement('canvas');
const context = canvas.getContext('2d');
canvas.height = viewport.height;
canvas.width = viewport.width;

await page.render({canvasContext: context, viewport: viewport}).promise;

4.4 CONVERSION PIPELINE
--------------------------------------------------------------------------------
PDF to DOCX:
1. Extract text runs with positioning
2. Map to HTML with style attributes
3. Convert HTML to DOCX via Mammoth
4. Package as .docx file

DOCX to PDF:
1. Parse DOCX XML structure
2. Extract paragraphs, styles, images
3. Reconstruct as PDF pages
4. Handle page breaks and margins

8. TROUBLESHOOTING
================================================================================

8.1 COMMON ISSUES
--------------------------------------------------------------------------------

PROBLEM: "File won't upload"
CAUSE: File too large or corrupted
SOLUTION: 
- Check file size (< 100MB recommended)
- Verify PDF is not corrupted (try opening in viewer)
- Refresh page and retry

PROBLEM: "Processing is slow"
CAUSE: Large file or low system resources
SOLUTION:
- Close other applications
- Reduce file size if possible
- Use more powerful device
- Split into smaller chunks

PROBLEM: "Download won't start"
CAUSE: Pop-up blocker or browser setting
SOLUTION:
- Disable pop-up blocker for site
- Check browser download settings
- Try different browser
- Manually trigger download again

PROBLEM: "Converted DOCX looks wrong"
CAUSE: Complex PDF formatting
SOLUTION:
- Simplify PDF before conversion
- Use "Preserve layout" option
- Manual adjustment may be needed
- Report specific issue with sample file

PROBLEM: "Pages appear blank"
CAUSE: Scanned PDF without OCR
SOLUTION:
- Tool processes PDF structure, not images
- Use OCR tool first to make text selectable
- Scanned pages are images, not text
